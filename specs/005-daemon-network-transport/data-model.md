# Data Model: Daemon Network Transport

## New Entities

### Credential
A bearer token issued by the daemon that authorizes a remote sigil-shell instance to connect.

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| id | string | non-empty, unique, alphanumeric + hyphens | Human-readable name (e.g. "macbook-pro") |
| token | string | starts with `sghl_`, 45 chars total | Bearer token; 40 hex chars of random entropy |
| token_hash | string | SHA-256 hex of token | Stored on disk instead of plaintext token |
| created_at | RFC3339 string | immutable | Timestamp of creation |
| revoked | bool | default false | Whether this credential has been revoked |
| revoked_at | RFC3339 string \| null | null until revoked | Timestamp of revocation |

**State transitions**: `active` → `revoked` (one-way; no un-revoke)

**Persistence**: `$XDG_DATA_HOME/sigil/credentials.json` — array of Credential objects (token_hash stored, never plaintext token). The plaintext token is only visible at generation time via `sigilctl credential add`.

**In-memory**: `map[string]*Credential` keyed by token hash, guarded by `sync.RWMutex`. Checked on every TCP connection auth.

### ServerTLSIdentity
The daemon's TLS certificate and private key, used to identify itself to connecting clients.

| Field | Type | Description |
|-------|------|-------------|
| cert_pem | string | PEM-encoded self-signed X.509 certificate (ECDSA P-256) |
| key_pem | string | PEM-encoded ECDSA private key |
| spki_fingerprint | string | `sha256:<base64>` hash of the Subject Public Key Info from the cert |
| generated_at | RFC3339 string | When the identity was created |
| valid_until | RFC3339 string | Cert expiry (1 year from generation) |

**Persistence**: `$XDG_DATA_HOME/sigil/server-cert.pem` and `$XDG_DATA_HOME/sigil/server-key.pem`. Loaded at startup; generated if absent.

**Rotation**: If cert expires, a new one is generated on next daemon start. All existing credentials become invalid (SPKI fingerprint changes) and must be regenerated. A warning is logged 30 days before expiry.

---

### NetworkConfig (config.go addition)
The TOML configuration block controlling the TCP listener.

| Field | TOML key | Type | Default | Description |
|-------|----------|------|---------|-------------|
| Enabled | enabled | bool | false | Whether to start the TCP listener |
| Bind | bind | string | "0.0.0.0" | IP address to listen on |
| Port | port | int | 7773 | TCP port |
| AllowedCredentials | allowed_credentials | []string | [] | Optional allowlist of credential IDs; empty = all valid tokens accepted |

---

### RemoteSession (in-memory only, not persisted)
An active authenticated connection from a remote sigil-shell.

| Field | Type | Description |
|-------|------|-------------|
| session_id | string | Random UUID assigned at auth |
| credential_id | string | ID of the credential used to authenticate |
| remote_addr | string | IP:port of the connecting client |
| connected_at | time.Time | When the session was established |

Tracked in a `sync.Map` on the Server. Used for logging and future session management (e.g., force-disconnect a session when a credential is revoked).

---

## Credential File (client-side, transferred out-of-band)
The JSON bundle generated by `sigilctl credential add` and copied to the host machine.

| Field | Type | Description |
|-------|------|-------------|
| id | string | Credential name, matches daemon-side Credential.id |
| token | string | Plaintext bearer token (only appearance in plaintext) |
| server_addr | string | `host:port` default connection target |
| server_cert_spki | string | `sha256:<base64>` SPKI fingerprint for TLS pinning |
| generated_at | RFC3339 string | Informational |

This file is **secret** — it contains the plaintext token and must be protected like an API key. On the host machine it lives at `$XDG_CONFIG_HOME/sigil-shell/daemon-credential.json` with mode `0600`.

## Config TOML Extension (sigild)

```toml
[network]
enabled = false
bind    = "0.0.0.0"
port    = 7773
# allowed_credentials = ["macbook-pro", "desktop"]  # optional allowlist
```

## sigil-shell Settings Extension (Tauri)

sigil-shell needs a way to store the remote connection config. This is persisted in the Tauri app's config directory as `daemon-settings.json`:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| transport | "unix" \| "tcp" | "unix" | Which transport to use |
| unix_socket_path | string \| null | null (auto) | Override for Unix socket path |
| tcp_credential_path | string \| null | null | Path to credential JSON file |
| tcp_addr_override | string \| null | null | Override `server_addr` from credential file |
